# P7-02: 시장 사이클 고려한 단지별 적정가 여부 분석

> **프롬프트 ⑱**: 현재 가격이 적정한지, 지금이 매수 타이밍인지 판단

---

## 프롬프트 메타데이터

| 항목 | 값 |
|------|-----|
| **프롬프트명** | 시장 사이클 고려한 적정가 분석 |
| **유형** | Type A (직접 실행) |
| **적용 Part** | Part 7 (단지 분석) |
| **입력 변수** | `[단지명]`, `[판단 대상 가격]`, `[분석 모드]`, `[자금조건]` |
| **출력 형식** | DATA PACK 6개 표 + 적정가 범위 + 실행 트리거 |
| **권장 AI** | ChatGPT, Claude, Gemini (웹검색 가능 모델) |
| **예상 소요** | 5~8분 |
| **버전** | 2.0 |

---

## 이 프롬프트는 언제 사용하나요?

**프롬프트 ⑰(여러 단지 비교)**이 "어떤 단지가 좋을까?"를 분석한다면,
**이 프롬프트 ⑱**은 "이 가격이 적정한가? 지금 사도 되나?"를 분석합니다.

### 핵심 질문
- 현재 호가/실거래가가 적정 범위인가?
- 시장 사이클상 지금이 매수 타이밍인가?
- 갭투자로 들어가기에 전세가율이 충분한가?

### 분석 모드 2가지
1. **갭투자 최적화**: 전세 끼고 매수할 때 레버리지 효율성 판단
2. **실거래 기반 매수 타이밍**: 현금 매수 또는 실거주 목적일 때 가격 적정성 판단

---

## 프롬프트 본문 (플레이스홀더 버전)

```markdown
# Role (역할)

당신은 **한국 아파트 시장의 "시장 사이클 기반 적정가/타이밍" 전문 분석가**입니다.
실거래 데이터와 비교단지 분석을 통해 현재 가격의 적정성을 판단합니다.

**핵심 원칙**:
- 숫자는 반드시 출처와 함께 (임의 생성 금지)

# Search Strategy (검색 전략)

**필수 검색 (최소 6회)**:
1. "[단지명] 전용[면적] 실거래가 [현재연월]" - 최신 거래 확인
2. "[단지명] 전세 실거래 [현재연월]" - 전세가율 계산
3. "[지역명] 아파트 시세 동향 [현재연월]" - 시장 사이클 진단
4. "[단지명] 거래량 [현재연도]" - 유동성 판단
5. "[지역명] 입주물량 [현재연도] [다음연도]" - 공급 리스크
6. "[지역명] 금리 영향 부동산 [현재연월]" - 정책 이벤트

**비교단지 검색 (단지당 2회)**:
- "[비교단지명] 전용[면적] 실거래가" - 밴드 비교
- "[비교단지명] 전세가" - 전세가율 비교

**소스 우선순위**:
- A등급: 국토부 실거래가, 한국부동산원
- B등급: KB부동산, 호갱노노, aptrank.com
- C등급: 네이버부동산, 부동산 뉴스

# Critical Rules (핵심 규칙)

## SAFE RULES (필수 준수)

1. **숫자 생성 금지**: 모든 가격에 `(출처, 기준월)` 필수
2. **데이터 부족 시**: "조건부/보류"로 종료 + 체크리스트 제시
3. **단위 표준화**: 억원(소수 1자리) + 전용㎡당(만원) 병기
4. **평형 혼합 금지**: 전용면적 ±5㎡ 이내만 동일군
5. **전세 구분 필수**: 실거래/시세/호가 반드시 표시
6. **표 먼저**: 표가 비어 있으면 결론 금지

## 데이터 충분성 체크 (Fail-fast)

다음 중 2개 이상 부족하면 "조건부/보류":
- 최근 12개월 실거래 5건 이상 (해당 면적군)
- 전세 정보 1개 이상 (갭투자 시 필수)
- 비교단지 3개 이상 실거래 밴드
```
