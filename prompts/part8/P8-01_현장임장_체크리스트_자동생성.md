# P8-01: 현장 임장 체크리스트 자동 생성

> **프롬프트 ㉑**: 단지 분석 결과를 바탕으로 A4 출력용 맞춤형 임장 체크리스트 생성

---

## 프롬프트 메타데이터

| 항목           | 값                                                            |
| -------------- | ------------------------------------------------------------- |
| **프롬프트명** | 현장 임장 체크리스트 자동 생성                                |
| **유형**       | Type A (직접 실행)                                            |
| **적용 Part**  | Part 8 (임장 및 의사결정)                                     |
| **입력 변수**  | `[단지 분석 리포트]`, `[비교 단지 2~3개]`, `[투자 목적]`      |
| **출력 형식**  | A4 출력용 체크리스트 (임장 개요 + 현장점검 + 인터뷰 + 평가표) |
| **권장 AI**    | ChatGPT, Claude, Gemini                                       |
| **예상 소요**  | 2~3분                                                         |
| **버전**       | 2.0                                                           |

---

## 프롬프트 본문 (플레이스홀더 버전)

```markdown
# Role (역할)

당신은 **10년 경력의 부동산 현장 분석 전문가**입니다.
매수자가 현장에 들고 가서 펜으로 직접 체크하고 메모할 수 있는 **'A4 출력용 임장 체크리스트'**를 작성합니다.

**핵심 원칙**:

- 분석 데이터에서 도출된 리스크와 단점을 현장 확인 항목으로 변환
- 흑백 인쇄에 최적화된 서식 (이모지 최소화, □/■ 기호 사용)
- 충분한 필기 공간 확보 (밑줄, 여백)

---

# Mission (임무)

[입력 데이터]의 분석 내용(리스크, 단점, 가격 이슈 등)을 반영하여:

1. 현장에서 반드시 눈으로 확인해야 할 항목 정리
2. 부동산 소장님께 질문할 핵심 사항 도출
3. 단지별 비교 평가표 제공
4. 종합 의견 기록란 구성

---

# 입력 데이터

## 단지 분석 리포트 요약

(여기에 Part 7에서 분석한 단지 리포트 또는 비교 분석 내용을 붙여넣으세요)

- 분석 단지: [단지명 1], [단지명 2], [단지명 3]
- 지역: [시/구/동]
- 관심 면적: 전용 [N]㎡
- 투자 목적: [시세차익 / 전세투자 / 실거주]

## 분석에서 도출된 주요 리스크/단점

- 입지 관련: [예: 역까지 도보 15분, 소음 우려]
- 상품성 관련: [예: 주차 1.0대, 구축 17년차]
- 가격 관련: [예: 호가와 실거래 괴리 15%]
- 수급 관련: [예: 2027년 인근 입주물량 3,000세대]

# 디자인 및 서식 지침 (Critical Rules)

1. **출력 최적화**: 이모지(Emoji) 사용 금지, 흑백 인쇄용 기호(-, ■, □) 사용
2. **필기 공간 확보**: 모든 질문 아래에 밑줄(`________`) 또는 여백 2~3줄
3. **가독성**: 각 섹션 사이에 구분선(`---`) 삽입
4. **체크박스**: 모든 확인 항목 앞에 `[ ]` 배치
5. **선택형 응답**: `□ 양호 □ 보통 □ 미흡` 형태로 체크 가능하게 구성

---

# Output Format (출력 형식)

## [헤더] 임장 개요

- 방문 일시, 날씨, 동행인, 방문 단지명 등을 적을 수 있는 상단 정보란

## 1. 외부/내부 현장 점검 (Physical Check)

### 1-1. 입지/접근성

- [입력 데이터]에서 도출된 '입지적 단점(역 거리, 소음 등)'을 확인하는 항목
- 각 항목에 체크 옵션과 메모 공간 제공

### 1-2. 단지 외부 환경

- 소음, 혐오시설, 상권 등 외부 환경 체크 항목

### 1-3. 단지 내부 (상품성)

- [입력 데이터]에서 도출된 '상품성 약점(동간 거리, 주차, 커뮤니티 등)' 확인 항목

## 2. 부동산 소장님 인터뷰 (Interview & Fact Check)

### 2-1. 가격/매물 관련

- 호가 vs 실거래 괴리 확인
- 급매물 존재 여부
- 최근 실거래 사례

### 2-2. 전세/수요 구조

- 전세 시세 및 수요 체감
- 전세 공실 기간

### 2-3. 리스크 체크

- 향후 공급 물량
- 하자/민원 이슈
- 실수요 vs 투자 수요 비중

## 3. 단지별 비교 평가표 (Scoring Table)

| 단지명 | 교통 | 쾌적성 | 생활인프라 | 가격매력 | 리스크 | 총평 |
| ------ | ---- | ------ | ---------- | -------- | ------ | ---- |
|        |      |        |            |          |        |      |

※ 각 항목: 1점(매우 나쁨) ~ 5점(매우 우수)

## 4. 종합 의견 (Review)

- 최우선 검토 단지
- 보류/탈락 단지
- 최종 총평
- 임장 후 추가 확인 필요 사항

---

**출력 시 주의**: A4 1~2페이지 분량으로 깔끔하게 정리해주세요.
```

## 사용법

### Step 1: 단지 분석 리포트 준비

Part 7에서 수행한 단지 분석 결과(여러 단지 비교, 적정가 분석 등)를 정리합니다.

### Step 2: 플레이스홀더 버전에 입력

**프롬프트 본문 (플레이스홀더 버전)**을 복사하여 `[...]` 부분을 본인 상황에 맞게 수정합니다:

- 분석 단지 2~3개
- 각 단지의 주요 특징, 장단점
- 분석에서 도출된 리스크/단점

### Step 3: AI에 입력

ChatGPT, Claude, Gemini 등에 붙여넣기하여 체크리스트를 생성합니다.

생성된 체크리스트를 워드/한글 파일에 붙여넣고 A4로 출력합니다.
