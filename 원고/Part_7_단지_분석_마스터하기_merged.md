# Part 7: 단지 분석 마스터하기

## "지역은 정했는데... 어떤 아파트를 사야 해요?"

'6개월 안에 신혼집을 구해야 하는' 정대리(31세)가 노트북 화면을 바라보며 한숨을 쉬었습니다.

Part 6에서 그는 드디어 **의정부 민락동**을 자신의 1순위 지역으로 선정했습니다. GTX-C 2030년 개통 예정, 신축 30평대가 예산 6억에 맞는 곳. 막연했던 "어디 살지?"가 "의정부 민락동"으로 구체화된 순간이었죠.

그런데 문제가 있습니다.

"민락동에만 아파트가... 몇 개야?"

네이버 부동산을 열어보니 민락동에만 수십 개의 단지가 있었습니다. 민락푸르지오, 민락센트럴자이, 민락휴먼시아, 민락두산위브... 이름만 봐도 머리가 아팠습니다.

"다 비슷비슷해 보이는데... 뭘 기준으로 골라야 하지?"

'서울 자가에 대기업 다녔던' 김부장(53세)이 고개를 끄덕였습니다.

"나도 노원 상계동 정했을 때 똑같은 고민 했어. 상계주공 1단지부터 12단지까지, 재건축 추진 단지까지... 대체 뭘 골라야 할지 감이 안 오더라고."

정대리가 답답한 표정으로 물었습니다.

"송과장님은 어떻게 하셨어요? 단지 하나하나 다 검색해보셨어요?"

'부동산 투자로 월급쟁이 부자된' 송과장(38세)이 웃으며 대답했습니다.

"그러다간 일주일이 걸려도 모자라. 나도 처음엔 그랬거든."

"그럼 어떻게요?"

송과장이 노트북을 열며 말했습니다.

"Part 6까지는 ChatGPT로 충분했어. 그런데 단지 분석부터는 **실시간 데이터**가 필요해. 실거래가, 호가, 전세가율... 이런 건 AI가 검색을 해야 알 수 있거든."

정대리가 고개를 갸웃거렸습니다.

"ChatGPT도 검색 기능 있지 않아요?"

"있긴 한데, 부동산 사이트를 직접 들어가서 데이터를 긁어오는 건 잘 못 해. 그래서 Part 7부터는 새로운 도구를 쓸 거야."

## "Comet이요? 그게 뭔데요?"

송과장이 브라우저를 열었습니다. 그런데 크롬이 아니었습니다.

"이게 **Comet**이야. Perplexity라는 AI 검색 회사에서 만든 브라우저인데, AI 비서가 브라우저 안에 내장되어 있어."

정대리가 화면을 들여다봤습니다.

"크롬이랑 뭐가 다른 거예요?"

송과장이 화이트보드에 비교표를 그렸습니다.

| 기능 | 크롬/엣지 | Comet |
|------|----------|-------|
| 웹 서핑 | O | O |
| AI 대화 | 별도 탭 필요 | **브라우저 내장** |
| 현재 페이지 분석 | X | **O (요약, 질문)** |
| 검색 기록 자연어 검색 | X | **O ("지난주 봤던 GTX 자료")** |
| 프로젝트별 정리 | X (탭만) | **O (Spaces 기능)** |
| 가격 | 무료 | **무료 (2025.10~)** |

김부장이 끼어들었습니다.

"나는 브라우저 바꾸는 거 귀찮아서 안 했는데... 꼭 써야 해?"

송과장이 고개를 저었습니다.

"Part 7 실습은 ChatGPT로도 가능해. 근데 Comet을 쓰면 **실습 ㉔, ㉕**에서 진짜 편해져. 네이버 부동산 페이지 열어놓고 AI한테 '이 단지 실거래가 정리해줘'라고 하면, AI가 그 페이지 데이터를 직접 읽어서 정리해주거든."

정대리의 눈이 반짝였습니다.

"진짜요? 일일이 복사-붙여넣기 안 해도 돼요?"

"응. 네이버 부동산, 호갱노노, 아실... 이런 사이트 열어놓고 AI한테 물어보면 돼."

"오른쪽 위에 '요약' 버튼 있어. 국토부 보도자료 50페이지짜리도 1분 만에 핵심만 뽑아줘."

"진짜 1분이요?"

"직접 해볼까?"

송과장이 국토부 3기 신도시 보도자료를 열었습니다. 53페이지짜리 PDF. 요약 버튼을 누르자 10초 만에 핵심 내용이 정리되었습니다.

```

- 대상 지역: 남양주 왕숙, 하남 교산, 인천 계양, 고양 창릉, 부천 대장

"1~2시간 걸릴 일이 10초로 줄어든 거네요?"

"그렇지. 물론 중요한 결정은 원문을 봐야 해. 하지만 '이 자료가 나한테 필요한지' 판단하는 건 요약만으로 충분해."

"ChatGPT는 별도 탭을 열어야 하잖아요. Comet은 **지금 보고 있는 페이지**를 AI가 함께 보면서 답해줘요."

송과장이 네이버 부동산에서 '민락푸르지오' 페이지를 열었습니다.

"여러 지역을 동시에 알아보면 정보가 섞이잖아. Spaces를 쓰면 프로젝트별로 분리할 수 있어."

## "근데 AI가 틀리면 어떡해요?"

정대리가 조심스럽게 물었습니다.

"송과장님, 근데 AI가 잘못된 정보 주면 어떡해요? 그거 믿고 집 샀다가..."

송과장의 표정이 진지해졌습니다.

"좋은 질문이야. **AI는 때때로 틀려**. 특히 실거래가, 전세가 같은 숫자는 반드시 확인해야 해."

김부장도 끼어들었습니다.

"내가 저번에 ChatGPT한테 상계주공 시세 물어봤더니 2023년 데이터를 주더라고. 지금이랑 1억 넘게 차이 나는 거야."

송과장이 표를 그렸습니다.

| AI 활용 | 신뢰도 | 검증 필요 |
|---------|--------|----------|
| 개념 설명 (LTV, DSR) | 높음 | 불필요 |
| 트렌드 분석 (상승/하락 방향) | 중간 | 최신 뉴스로 확인 |
| **실거래가, 호가** | **낮음** | **반드시 공식 사이트 확인** |
| 정책 규제 수치 | 중간 | 정부 사이트로 확인 |

"그래서 Part 7 실습에서는 **검증 체크포인트**를 꼭 넣을 거야. AI가 준 숫자를 어디서 확인해야 하는지 알려줄게."

정대리가 안심한 표정으로 말했습니다.

"그러면 AI는 '1차 정리용'으로 쓰고, 최종 확인은 직접 하는 거네요."

"정확해. **빠른 정리는 AI, 최종 확인은 공식 사이트**. 이 원칙만 지키면 돼."

```

| 실습 | 질문 | 산출물 |
|------|------|--------|

| ㉕ 수익률 분석 | 지금이 사야 할 타이밍인가? | 매수 타이밍 판단 |

김부장이 끼어들었습니다.

"나는 노원 상계동으로 같은 분석 해볼게. 재건축 추진 단지들 비교해봐야겠어."

송과장이 웃으며 말했습니다.

"좋아요. 세 명이 각자 다른 지역으로 실습하면 비교도 재밌을 거예요."

## 이번 Part에서 할 4가지 실습

정대리가 기대감 어린 표정으로 물었습니다.

"4개 실습 미리보기 해주세요!"

### 실습 ㉒: 단지 심층 비교 분석

> "민락푸르지오 vs 민락센트럴자이 vs 민락휴먼시아, 뭘 살까?"

- 5가지 평가지표: 상품성, 입지, 수급, 가격, 리스크
- Self-Consistency 3회 분석으로 신뢰도 확보
- **산출물**: 최적 단지 1개 선정

### 실습 ㉓: 적정가 여부 분석

> "민락푸르지오 84㎡가 5.2억인데... 비싼 거야, 싼 거야?"

- 최근 12~24개월 실거래가 분석
- 시장 사이클 고려
- **산출물**: 적정가 범위 + 매수/보류/분할 판정

### 실습 ㉔: 실거래가 vs 호가 비교 (Comet 활용)

> "호가 5.5억, 실거래 5.2억... 협상하면 얼마까지?"

- Comet으로 네이버 부동산 실시간 데이터 수집
- 호가-실거래 차이율 분석
- **산출물**: 협상 목표가 + 시장 온도

### 실습 ㉕: 과거 수익률 분석 (Comet 활용)

> "2022년에 샀으면 지금 얼마 벌었을까?"

- 연도별 매매가/전세가 추이 분석
- 2년 보유 기준 수익률 계산
- **산출물**: 저점/고점 타이밍 + 현재 매수 판단

# 실습 ㉓: 적정가 여부 분석

실습 ㉒에서 정대리는 드디어 **민락푸르지오**를 1순위 단지로 선정했습니다. 5가지 평가지표로 비교 분석하고, Self-Consistency로 검증까지 마쳤죠.

'부동산 투자로 월급쟁이 부자된' 송과장(38세)이 끼어들었습니다.

## 5가지 분석 요소

송과장이 화이트보드에 적었습니다.

"적정가 분석할 때 이 5가지를 봐야 해."

정대리가 물었습니다.

"5가지요? 뭔데요?"

| 분석 요소 | 내용 | 왜 중요한가? |
|----------|------|-------------|
| **1. 실거래 데이터** | 최근 12~24개월 거래 내역 | 실제로 얼마에 팔렸는지가 진짜 가격 |
| **2. 전세 정보** | 전세가, 전세가율 | 갭투자 효율성 판단 (60% 이상이면 좋음) |
| **3. 비교단지** | 주변 비슷한 단지 가격 | 상대적으로 비싼지 싼지 비교 |
| **4. 거래량** | 월별 거래 건수 | 시장 심리 파악 (거래절벽 vs 회복) |
| **5. 정책/이벤트** | 금리, 규제, 대규모 입주 | 가격에 영향 줄 변수들 |

김부장이 끼어들었습니다.

"전세가율이 60% 이상이면 좋다고? 왜?"

송과장이 대답했습니다.

"전세가율이 높으면 **내 돈을 적게 넣고 투자**할 수 있어. 예를 들어..."

```
매매가 10억, 전세가 6억 (전세가율 60%)
→ 내 돈 4억으로 10억짜리 아파트 투자

매매가 10억, 전세가 4억 (전세가율 40%)
→ 내 돈 6억 필요 → 자금 효율 떨어짐
```

정대리가 고개를 끄덕였습니다.

"아... 전세가율이 낮으면 갭투자하기 어렵구나."

## Self-Consistency: 왜 5번 분석할까?

정대리가 물었습니다.

"왜 5번이나 분석해요? 한 번이면 안 돼요?"

송과장이 표를 그렸습니다.

| Run | 분석 관점 | 초점 |
|-----|----------|------|
| **Run 1** | 비교단지 중심 | 주변 단지 대비 가격 적정성 |
| **Run 2** | 전세/레버리지 중심 | 갭투자 효율성 |
| **Run 3** | 거래량 심리 중심 | 시장 분위기, 거래 활성화 |
| **Run 4** | 공급 충격 중심 | 대규모 입주 영향 |
| **Run 5** | 정책/규제 중심 | 규제 변화, 금리 영향 |

"AI도 **한 번의 분석에서 실수**할 수 있어. 근데 5가지 다른 관점에서 보면?"

정대리가 대답했습니다.

"아... 4개가 같은 결론이면 그게 맞을 가능성이 높고, 결론이 다 다르면 뭔가 불확실한 거네요?"

"정확해. 그게 **Self-Consistency의 핵심**이야."

김부장이 끼어들었습니다.

"근데 5번 질문하면 토큰 많이 쓰는 거 아니야?"

"응. 그래서 이 프롬프트는 **한 번에 5회 분석을 다 시켜**. 효율적이지."

### 플레이스홀더 버전

```markdown
당신은 부동산 데이터 분석 전문가입니다. 사용자가 지정한 아파트 단지에 대해, 아실의 실거래가와 네이버부동산 호가를 비교 분석하고 최근 거래가격·거래량까지 반영해 인사이트를 도출하세요.

1. **데이터 수집**:
   - 아실(asil.kr)에서 해당 단지의 최근 12개월 매매 실거래가를 조회합니다.
     - 사용자가 지정한 전용면적에 대해서만 우선 분석합니다.
     - 거래일, 거래가격(억 단위), 층수, 동 정보를 표로 정리합니다.
   - 네이버부동산에서 동일 단지의 현재 매물 호가를 조회합니다.
     - 매물수, 최저 호가, 최고 호가, 평균 호가를 표로 정리합니다.

2. **비교 분석**:
   | 평형(전용) | 실거래가 (최근 3개월 평균) | 실거래가 (최근 3건 평균) | 호가 (평균/범위) | 호가-실거래 차이율(%) | 해석 |

   - 차이율(%) = (호가 평균 - 실거래가 평균) / 실거래가 평균 × 100

   **최근 거래량·거래 추이 요약**:
   - 최근 3~6개월 월별 거래량(건수)을 표로 정리
   - 거래량 증가 + 가격 상승: 상승장 신호
   - 거래량 감소 + 가격 하락: 하락장 신호
   - 거래량 감소 + 가격 횡보: 거래 절벽

3. **투자 인사이트(체크리스트)**:
   - ✅ 호가 > 실거래 15% 이상: 협상 여지 큼
   - ✅ 최근 신고가 돌파 여부
   - ❌ 거래량 급감: 유동성 위험
   - ❌ 급매 다수 출현: 하방 리스크

4. **3가지 시나리오 제시**:
   - 기본 시나리오: 현재 추세 유지 시 6개월 후 예상가
   - 상향 시나리오: 거래량 증가 + 가격 상승 시 예상가
   - 하향 시나리오: 거래량 감소 + 가격 하락 시 예상가

**분석할 아파트 단지 및 평형:
[단지명] / [전용면적]**
```

## 매물 선택 기준

**좋은 매물 조건**

| 조건 | 이유 | 정대리 기준 |
|------|------|-----------|
| **중고층 (10~20층)** | 조망 + 유동성 | O |
| **남향/남동향** | 채광 + 선호도 | O |
| **호가 ≤ 실거래+5%** | 협상 여지 | 5.8억 이하 |
| **급매 아님** | 하자 위험 | 확인 필요 |

**피해야 할 매물**

| 조건 | 이유 |
|------|------|
| **저층 (1~3층)** | 유동성 낮음, 프라이버시 |
| **북향** | 채광 불리, 선호도 낮음 |

## Key Takeaway

1. **실거래가 vs 호가**: 실거래가는 "진짜 시세", 호가는 "희망 시세"
2. **호가-실거래 차이율**: 15% 이상이면 협상 여지 큼, 5% 이하면 깎기 어려움
3. **거래량 추이**: 거래량 증가+가격 상승=상승장, 거래량 감소+가격 하락=하락장
4. **데이터 기반 협상**: "최근 실거래가 X억인데요"가 강력한 협상 근거
5. **매물 선택 기준**: 중고층, 남향/남동향, 호가 ≤ 실거래+5%

## 갭투자란?

정대리가 고개를 갸웃거렸습니다.

"근데 수익률을 어떻게 계산해요? 집값이 오른 만큼이 수익인 거 아니에요?"

송과장이 화이트보드에 적었습니다.

"**갭투자**를 알아야 해. 부동산 투자에서 가장 많이 쓰는 방법이야."

```
갭투자란?
= 전세를 끼고 아파트를 사는 것

예시:
매매가 10억, 전세가 6억
→ 내 돈(실투자금) = 10억 - 6억 = 4억

2년 후 매매가 12억이 되면?
→ 수익 = 12억 - 10억 = 2억
→ 수익률 = 2억 ÷ 4억 = 50%
```

김부장이 물었습니다.

"아파트가 20% 올랐는데 수익률은 50%야?"

"응. 그게 **레버리지 효과**야. 전세를 끼면 내 돈을 적게 넣으니까 수익률이 높아져."

정대리가 고개를 끄덕였습니다.

"아... 그래서 갭투자가 인기구나. 적은 돈으로 큰 수익을 낼 수 있으니까."

"맞아. 근데 **리스크도 있어**. 집값이 떨어지면 손실도 레버리지만큼 커지거든."

## Part 7 핵심 인사이트

### 1. 같은 동네라도 단지마다 다르다

민락동에만 수십 개 단지가 있지만, 5가지 평가지표로 분석하면 확실한 1순위가 나옵니다. **"다 비슷해 보인다"는 착각**을 깨는 것이 단지 분석의 핵심입니다.

### 2. 가격은 사이클 위치로 판단한다

같은 5억이라도 고점에서는 비싸고, 저점에서는 싼 것입니다. **"절대 가격"이 아닌 "상대 위치"**로 적정가를 판단해야 합니다.

### 3. 데이터가 협상력이다

"최근 실거래가 5.6억인데요"라고 말할 수 있으면 협상이 유리해집니다. **AI로 데이터를 정리하고, 공식 사이트로 검증**하는 습관이 중요합니다.

### 4. 타이밍이 수익률의 핵심

2021년 고점 매수 vs 2022년 저점 매수, 같은 단지에서 100%p 이상 수익률 차이가 납니다. **"언제 사느냐"가 "어디 사느냐"만큼 중요**합니다.

## 이제 진짜 집 보러 갑니다

정대리는 노트북을 덮고 일어섰습니다.

```
정대리의 Part 7 결과 요약
- 단지: 민락푸르지오 84㎡
- 목표가: 5.5억 (실거래 기준)
- 판단: 적정가 범위 내, 분할접근 권장
- 다음: 임장으로 현장 확인
```

예전 같았으면 "일단 가보자"며 아무 준비 없이 현장에 갔을 겁니다. 그런데 이제는 다릅니다. **"이 단지의 역 거리가 진짜 10분인지, 소음이 심한지, 5.5억까지 협상 가능한지"** 확인할 포인트가 명확합니다.

그리고 그 확인이 **막연한 불안이 아닌 데이터 기반 검증**이라는 게 달라진 점입니다.

"이제 Part 8에서 직접 가서 확인하면 돼. AI가 체크리스트 만들어주고, 질문 리스트 만들어주고... 현장에서 놓치는 것 없이 볼 수 있게 도와줄 거야."

Part 7에서 '어떤 단지, 어떤 가격'을 분석했다면, **Part 8에서는 '진짜 이 집이 맞는지' 현장에서 확인합니다.**

노트북을 가방에 넣은 정대리는 설렘 반, 긴장 반으로 Part 8 첫 실습을 열었습니다.
